How a Machine Learns: Gradient Descent From Scratch

Link to Github Repo

Ever wondered how machines *actually learn*? In this blog, we break down one of the core engines behind modern machine learning — Gradient Descent. No libraries, no shortcuts. Just pure math, logic, and Python.



Why This Matters

  • - It demystifies how ML models optimize themselves.
  • - Helps you understand what's behind algorithms like Linear Regression, Logistic Regression, and Neural Networks.
  • - Reinforces your grasp of core math concepts like derivatives, slope, and minimization.

The Story

Imagine a blindfolded hiker on a mountain trying to reach the lowest point in the valley. He can’t see the whole terrain, but by feeling the steepness under his feet, he decides which direction to move. This is exactly how a machine *"learns"* — by descending the slope of errors, step by step, adjusting parameters to minimize loss.

In our case, the valley is the Loss Function, and the slope is its gradient. Every step we take is an epoch, and how large our steps are is defined by the learning rate.

What Are We Teaching the Machine?

Say we want to predict how much a customer will spend based on how much time they spend on our site.

The machine tries to learn this relationship:

spending = weight_1 × time + weight_0

The weights (also called parameters) are like knobs the machine adjusts to improve its guesses.

At first, these knobs are wrong. So the machine guesses, sees how wrong it was, and then adjusts the weights — again and again.



MSE: Measuring the Wrongness

To know how wrong it is, the machine uses a formula: Mean Squared Error (MSE).

MSE = average of (prediction − actual)2

  • If all predictions are perfect, MSE = 0.
  • Bigger mistakes are punished more (they’re squared).

This MSE becomes the “height” on the mountain. Lower is better.

Learning Rate: Step Size

The learning rate decides how big each step is.

  • Too big? You might overshoot and miss the minimum.
  • Too small? You’ll take forever to get there.

A good starting value is 0.1. This tells the machine to step cautiously but effectively.



Epochs: Repetition Matters

One epoch = the machine has seen all the data once.

Usually, we do this 50–100 times so it can gradually learn better.

Mini-Batches: Study Groups

Feeding all data at once is slow. One-by-one is chaotic.

Mini-batches (e.g. 20 samples at a time) offer balance.

Like studying in small groups — fast and effective.

Why Shuffle?

What if we feed only 2020 data first, then 2021?

The machine might learn bad or outdated patterns.

Shuffling the data before every epoch avoids this, helping the machine learn fairer rules.



Full Code: Gradient Descent From Scratch

Here’s a full Python implementation, complete with comments. No frameworks, just NumPy:


import numpy as np

# Step 1: Generate synthetic data
np.random.seed(0)
X = 2 * np.random.rand(100, 1)             # 100 data points, one feature (time)
y = 4 + 3 * X + np.random.randn(100, 1)    # True relation: y = 4 + 3x + noise

# Step 2: Add a bias (intercept) term to input
X_b = np.c_[np.ones((X.shape[0], 1)), X]   # Shape: (100, 2)

# Step 3: Initialize weights randomly
theta = np.random.randn(2, 1)              # Two weights: [intercept, slope]

# Step 4: Set hyperparameters
learning_rate = 0.1
n_epochs = 50
batch_size = 20
m = X_b.shape[0]                           # Total number of samples

# Step 5: Define the loss function (Mean Squared Error)
def compute_loss(X, y, theta):
    predictions = X.dot(theta)
    errors = predictions - y
    return np.mean(errors ** 2)

# Step 6: Train using mini-batch gradient descent
for epoch in range(n_epochs):
    # Shuffle data at the start of each epoch
    indices = np.random.permutation(m)
    X_b_shuffled = X_b[indices]
    y_shuffled = y[indices]
    
    for i in range(0, m, batch_size):
        # Create mini-batch
        X_batch = X_b_shuffled[i:i+batch_size]
        y_batch = y_shuffled[i:i+batch_size]
        
        # Predict and compute gradients
        predictions = X_batch.dot(theta)
        errors = predictions - y_batch
        gradients = (2 / batch_size) * X_batch.T.dot(errors)
        
        # Update weights
        theta = theta - learning_rate * gradients

    # Print loss after each epoch
    loss = compute_loss(X_b, y, theta)
    print(f"Epoch {epoch+1}, Loss: {loss:.4f}")

# Final weights after training
print("\nFinal learned weights (theta):")
print(theta)

Recap for Everyone

Concept What it Means
Weights The values the machine adjusts to get better at prediction
Loss (MSE) A number showing how wrong the machine's predictions are
Gradient Descent The process of stepping downhill to reduce error
Learning Rate The size of each step the machine takes during learning
Epoch One complete pass through all training data
Mini-Batch Small groups of data used to speed up training
Shuffle Mixes data each round to prevent pattern bias
